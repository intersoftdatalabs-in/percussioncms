E2 implements JDBC drivers for any non-JDBC data stores it wants to access as SQL data stores. E2 does not implement fully JDBC compliant drivers. This is primarily for speed of development as we do not need the entire JDBC functionality set.

Implementation Phases

Phase I:
  • ODBC IPSDriverMetaData support
  • File System JDBC Driver with catalog only support (for XML)
Phase III:
  • File System JDBC Driver with transaction support (for XML)
  • Full XML JDBC Driver (extends File Systen JDBC driver)
